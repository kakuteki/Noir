Accessibility

Noir includes built-in accessibility features. Follow these guidelines for WCAG 2.1 AA compliance.

Component Required Attributes Notes
Modal (dialog) aria-labelledby Use native <dialog> element. Browser provides Esc-close and focus trap via showModal().
Dropdown aria-haspopup, aria-expanded, role="menu" Toggle aria-expanded with JS. Add role="menuitem" to items.
Toast role="status", aria-live="polite" Use role="alert" for error/warning toasts. Container needs aria-live.
Accordion Native <details>/<summary> Browser handles expand/collapse accessibility natively.
Forms for/id, aria-describedby Link labels to inputs with for. Use aria-describedby for hints/errors.
Screen reader .sr-only Visually hide content while keeping it accessible to assistive technology.
Reduced motion Automatic All animations and transitions respect prefers-reduced-motion: reduce.
Focus Automatic All interactive elements show :focus-visible outline. Use .focus\:ring-* for custom rings.
